# `submission/` — Day 6 deliverables

This folder holds everything you need to submit Overheard to the hackathon. These are drafts — edit to your voice before posting.

| File | For |
|---|---|
| [`DESCRIPTION.md`](./DESCRIPTION.md) | ~250-word text description for the submission form |
| [`KIRO_USAGE.md`](./KIRO_USAGE.md) | ~750-word Kiro-usage write-up covering all six sub-criteria |
| [`VIDEO_SCRIPT.md`](./VIDEO_SCRIPT.md) | 60-second storyboard + production notes + platform-cut specs |
| [`SOCIAL_POSTS.md`](./SOCIAL_POSTS.md) | Drafts for X, LinkedIn, Instagram, TikTok, and community outreach |
| [`CHECKLIST.md`](./CHECKLIST.md) | Pre-submit checklist — run through before hitting submit |

## Day 6 order of operations

1. **Morning (by 10:00)** — run the `CHECKLIST.md` once. Fix anything red.
2. Copy `DESCRIPTION.md` + `KIRO_USAGE.md` into the submission form fields.
3. Paste the YouTube video URL into the video field.
4. Paste the public GitHub repo URL.
5. Post all four social posts using `SOCIAL_POSTS.md` as templates.
6. Submit the form (well before 17:00).
7. Post the community-vote message in one or two Discord/Slack channels you're already a member of.

Buffer 4–6 hours before the deadline. Do not submit at 16:55.
